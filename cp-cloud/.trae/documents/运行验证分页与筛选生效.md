## 验证目标
- 启动 system 服务并以 dev-local 模式运行。
- 请求 `/system/user/list` 携带 `pageNum/pageSize` 与时间范围，确认 SQL 出现 `COUNT(*)` 与 `LIMIT`，且响应 `rows` 仅当前页，`total` 为真实总数。
- 验证当 `beginTime/endTime` 为 `undefined` 时被忽略。

## 执行步骤
1) 启动服务：使用本地数据源与 `--spring.profiles.active=dev-local`，关闭 Nacos 依赖。
2) 发送请求：
- `http://localhost:9220/system/user/list?pageNum=1&pageSize=10`
- `http://localhost:9220/system/user/list?pageNum=1&pageSize=10&params[beginTime]=2025-11-01 00:00:00&params[endTime]=2025-11-21 23:59:59`
- `http://localhost:9220/system/user/list?pageNum=1&pageSize=10&params[beginTime]=undefined&params[endTime]=undefined`
3) 观察控制台日志：检索是否出现 `SELECT COUNT(*)` 与 `LIMIT` 以及 `create_time` 的过滤条件。
4) 校验响应：
- 第一条请求：`rows` 长度应为 10，`total` 等于真实总记录数。
- 第二条请求：`rows` 为时间范围内数据的分页，`total` 为范围内总数。
- 第三条请求：与第一条一致。

## 交付
- 返回日志与响应片段，确认修复有效。
