## 要解决的问题
- 前端携带 `pageNum/pageSize` 但接口返回“全部数据”或筛选不生效。
- SQL 日志只见一条查询且无 `COUNT(*)`/`LIMIT`，以及没有时间范围筛选条件。

## 改进方案
1) 增加时间范围筛选：
   - 在 `SysUserServiceImpl.selectUserPage` 读取 `user.getParams().get("beginTime")`、`user.getParams().get("endTime")`；
   - 忽略值为 `null/空字符串/"undefined"` 的参数；
   - 使用 `qw.ge(SysUser::getCreateTime, beginDate)` 与 `qw.le(SysUser::getCreateTime, endDate)` 进行过滤（日期解析用已有 `DateUtils.parseDate`）。
2) 保持已修复的分页拦截器与总数兜底：
   - `ApplicationConfig` 标注为配置类，确保 MP 拦截器注册；
   - `selectUserPage` 已添加 `selectCount(qw)` 兜底填充 `total`。
3) 验证与回归：
   - 重编译并启动 system；
   - 访问 `/system/user/list?pageNum=1&pageSize=10&params[beginTime]=...&params[endTime]=...`；
   - 日志应出现 `COUNT(*)` 与 `LIMIT`，且 SQL 包含 `create_time` 时间过滤；
   - 响应 `rows` 仅为当前页数据，`total` 为真实总数。

## 影响范围
- 仅改动 `cp-modules/cp-system/src/main/java/com/cp/system/service/impl/SysUserServiceImpl.java`，不更改接口与返回结构。
